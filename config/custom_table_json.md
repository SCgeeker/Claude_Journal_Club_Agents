# JSON 表格處理指引

本檔案指導 LLM 如何處理已轉換為 JSON 格式的學術論文表格。

---

## JSON 表格格式

表格會以下列 JSON 結構呈現：

```json
{
  "table_id": "Table 1",
  "caption": "表格標題",
  "headers": ["欄位1", "欄位2", "欄位3"],
  "rows": [
    {"欄位1": "值1", "欄位2": "值2", "欄位3": "值3"},
    {"欄位1": "值4", "欄位2": "值5", "欄位3": "值6"}
  ],
  "footnotes": ["* p < 0.05", "** p < 0.01"]
}
```

---

## 處理優勢

JSON 格式的優點：
- **結構化存取**：可精確定位特定欄位或數值
- **程式處理友善**：便於後續自動化分析
- **無格式歧義**：避免 Markdown 表格的對齊問題

---

## 分析要點

### 統計表格

當表格包含統計數據時，請關注：

1. **效果量**（Effect Size）
   - Cohen's d, η², r 等
   - 判斷效果大小（小/中/大）

2. **顯著性**
   - p 值分布
   - 多重比較校正（如 Bonferroni）

3. **信賴區間**
   - 95% CI 的寬度
   - 是否跨越零點

### 描述性統計表

1. **集中趨勢**：M, Mdn
2. **離散程度**：SD, IQR, Range
3. **樣本量**：N, n

### 迴歸分析表

1. **係數方向**：正/負
2. **標準化係數**：β vs. B
3. **模型適配度**：R², Adjusted R²

---

## 輸出格式

請以下列格式摘要表格內容：

```markdown
### Table X: [標題]

**研究設計**：[實驗/調查/後設分析]

**主要發現**：
- [發現 1：數值支持]
- [發現 2：數值支持]

**統計細節**：
- 樣本量：N = [數值]
- 主要效果：[統計量] = [數值], p [符號] [數值]
- 效果量：[指標] = [數值]

**解讀**：
[1-2 句話說明這些數據的意涵]
```

---

## 範例

### 輸入 JSON

```json
{
  "table_id": "Table 2",
  "caption": "Experiment 1: Response times by condition",
  "headers": ["Condition", "M (ms)", "SD", "95% CI"],
  "rows": [
    {"Condition": "Congruent", "M (ms)": "423", "SD": "67", "95% CI": "[408, 438]"},
    {"Condition": "Incongruent", "M (ms)": "512", "SD": "83", "95% CI": "[493, 531]"}
  ],
  "footnotes": ["Note: N = 48 per condition"]
}
```

### 輸出摘要

```markdown
### Table 2: Experiment 1: Response times by condition

**研究設計**：受試者內設計實驗

**主要發現**：
- 不一致條件的反應時間（M = 512ms）顯著長於一致條件（M = 423ms）
- 差異為 89ms，約 21% 的增幅

**統計細節**：
- 樣本量：N = 48 per condition
- 一致條件：M = 423ms, SD = 67, 95% CI [408, 438]
- 不一致條件：M = 512ms, SD = 83, 95% CI [493, 531]

**解讀**：
信賴區間未重疊，顯示條件間存在穩健的差異，支持認知衝突假說。
```

---

## 注意事項

1. **保留原始數值**：摘要時引用表格中的實際數字
2. **標註單位**：ms, %, Hz 等
3. **區分顯著與否**：依據 p 值和信賴區間判斷
4. **避免過度解讀**：僅報告表格直接呈現的資訊

# 🎨 學術風格定義

本目錄包含學術風格的詳細定義和配置，用於指導內容生成的風格、語氣和結構。

---

## 📁 文件說明

### academic_styles.yaml

**用途**: 學術風格的結構化定義

**基於**: SciMaker Journal Club 的風格系統，提取並結構化為 YAML 配置

**結構**:
```yaml
styles:
  style_name:
    name: "風格名稱"
    description: "風格描述"
    characteristics:
      - 特徵1
      - 特徵2
    use_cases:
      - 使用場景1
      - 使用場景2
    tone: "語氣描述"
    emphasis: "重點領域"
```

## 🎨 風格詳細說明

### 1. classic_academic（經典學術）

**特徵**:
- 傳統學術語言和格式
- 強調理論框架和方法論
- 嚴謹的論述結構
- 引用充分且正式

**語氣**: 正式、客觀、嚴謹

**強調**: 理論、方法、批判性分析

**適用場景**:
- 正式研討會報告
- 學位論文答辯
- 學術期刊投稿
- 資深學者聽眾

**範例結構**:
```
1. 研究背景與理論框架
2. 文獻回顧與批判
3. 研究方法與設計
4. 結果與數據分析
5. 討論與理論貢獻
6. 結論與未來研究方向
```

---

### 2. modern_academic（現代學術）⭐ 默認

**特徵**:
- 結合視覺化和數據呈現
- 清晰易懂的表達
- 平衡深度與可讀性
- 適度使用圖表和範例

**語氣**: 專業但易懂、現代、動態

**強調**: 清晰溝通、數據視覺化、實用性

**適用場景**:
- 會議報告
- 工作坊簡報
- 跨領域交流
- 多元聽眾背景

**範例結構**:
```
1. 研究問題與動機（配圖）
2. 核心概念解析
3. 研究方法（流程圖）
4. 主要發現（數據視覺化）
5. 實務啟示
6. 未來展望
```

- 醫學簡報

**範例結構**:

4. 治療選項與證據

### 8. zettelkasten（卡片盒筆記）✨ NEW

**特徵**:
- 原子化筆記（每張卡片一個概念）
- 強調連結性和網絡結構
- 語義化 ID 系統
- 支援永久筆記生成

**語氣**: 簡潔、連結導向、知識網絡

**強調**: 原子性、連結語義、知識網絡

**適用場景**:
- 知識管理
- 永久筆記系統
- 概念網絡建構
- 跨文獻知識整合

**卡片類型**:
1. **concept**（概念卡）: 定義核心概念
2. **method**（方法卡）: 說明研究方法或技術
3. **finding**（發現卡）: 記錄重要發現或證據
4. **question**（問題卡）: 提出未解問題或思考方向

**連結語義**:
- **基於**: A 概念建立在 B 概念之上
- **導向**: A 概念引申出 B 概念
- **相關**: A 概念與 B 概念相關
- **對比**: A 概念與 B 概念對比
- **上位**: A 是 B 的上位概念
- **下位**: A 是 B 的下位概念

**範例結構**:
```
zettel_index.md:
  - 卡片列表（按類型分類）
  - 概念網絡圖（Mermaid）
  - 標籤索引

zettel_cards/:
  - 領域-日期-序號.md
  - 標題、類型、核心概念
  - 描述、內容、連結網絡
  - 來源脈絡、AI 筆記、人類筆記
```

## 📊 風格選擇指南

### 根據聽眾選擇

| 聽眾類型 | 推薦風格 | 理由 |
|---------|---------|------|
| 資深學者 | classic_academic | 符合學術傳統 |
| 一般研究者 | modern_academic ⭐ | 平衡深度與易讀 |
| 臨床醫師 | clinical | 強調應用 |
| 跨領域聽眾 | teaching | 易懂且友善 |
| 方法學專家 | research_methods | 技術細節充分 |
| 文獻回顧需求 | literature_review | 系統性整理 |
| 實務工作者 | case_analysis | 實務導向 |
| 知識管理 | zettelkasten | 原子化且可連結 |

### 根據場合選擇

| 場合 | 推薦風格 | 理由 |
|------|---------|------|
| 正式研討會 | classic_academic | 學術規範 |
| 一般會議 | modern_academic ⭐ | 清晰有效 |
| 工作坊 | teaching 或 case_analysis | 互動學習 |
| 論文答辯 | classic_academic | 嚴謹完整 |
| 方法訓練 | research_methods | 技術深入 |
| 文獻報告 | literature_review | 系統綜合 |
| 臨床會議 | clinical | 實證應用 |
| 個人學習 | zettelkasten | 知識累積 |

## 🔧 自定義風格

### 新增風格步驟

1. **定義風格** (`academic_styles.yaml`):
   ```yaml
   my_custom_style:
     name: "我的自定義風格"
     description: "風格說明"
     characteristics:
       - 特徵1
       - 特徵2
     use_cases:
       - 使用場景
     tone: "語氣"
     emphasis: "重點"
   ```

2. **更新 Prompt 模板** (`templates/prompts/journal_club_template.jinja2`):
   - 添加對應的風格指引
   - 定義特定的格式要求

3. **測試風格**:
   ```bash
   python make_slides.py "測試主題" --pdf test.pdf --style my_custom_style
   ```

4. **文檔更新**:
   - 更新本 README
   - 更新 CLAUDE.md
   - 提供範例輸出
